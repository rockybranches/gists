# generate-terrain-app

Standalone Electron desktop application for interactive 3D terrain generation.  
**No Python backend required** — terrain generation is implemented entirely in TypeScript.

---

## Overview

`generate-terrain-app` is a self-contained Electron GUI that lets you generate and explore parameterised pseudorandom 3D terrain in real time. All noise/heightmap computation and STL mesh generation are implemented as a native TypeScript module (`src/terrain.ts`) compiled directly into the app bundle — there is **no dependency on the Python backend** (`generate-terrain.py`).

The interactive 3D surface is rendered using [Plotly.js](https://plotly.com/javascript/) loaded from CDN.

---

## Architecture

```
generate-terrain-app/
├── src/
│   └── terrain.ts        ← TypeScript terrain engine (heightmap, STL)
├── dist/
│   └── terrain.js        ← Compiled output (generated by tsc, gitignored)
├── main.js               ← Electron main process — IPC, STL export dialog
├── preload.js            ← Context-isolated IPC bridge
├── renderer.js           ← Plotly 3D surface rendering, UI interactions
├── index.html            ← App shell + Plotly CDN script tag
├── tsconfig.json         ← TypeScript compiler config
├── package.json          ← npm dependencies + build scripts
└── flake.nix             ← Nix dev shell (Node.js + Electron)
```

### Terrain Engine (`src/terrain.ts`)

A faithful TypeScript port of the Python backend algorithms:

| Feature | Python (original) | TypeScript (this app) |
|---|---|---|
| PRNG | `np.random.seed(n)` | Mulberry32 PRNG |
| Upsampling | `scipy.ndimage.zoom` | Bilinear interpolation |
| Octave FBM noise | ✅ | ✅ |
| Normalisation 0→1 | ✅ | ✅ |
| Closed manifold mesh | ✅ | ✅ |
| Binary STL writer | ✅ | ✅ |

## Development Environment

### Option A — Nix dev shell (recommended)

Provides a reproducible Node.js + Electron environment:

```bash
cd generate-terrain-app
nix develop        # enter the shell
npm install
npm start
```

### Option B — System Node.js

Requires Node.js ≥ 18 on your `$PATH`:

```bash
cd generate-terrain-app
npm install
npm start
```

## Parameters

| Parameter | Range | Description |
|---|---|---|
| Width | 20 – 300 | X-axis grid resolution |
| Length | 20 – 300 | Y-axis grid resolution |
| Height Scale | 1 – 100 | Peak elevation multiplier |
| Feature Scale | 5 – 200 | Horizontal feature wavelength |
| Octaves | 1 – 10 | FBM noise detail layers |
| Roughness | 0.1 – 1.0 | Amplitude decay per octave |
| Seed | 0 – 9 999 999 | Reproducible PRNG seed |

## STL Export

Click **Export STL** after generating terrain. A native save dialog will appear.  
The exported `.stl` file is a closed, manifold binary STL mesh with a flat solid base — suitable for 3D printing.
